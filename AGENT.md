## 1. Project Identity

**DeepClaw UI** is a real-time web dashboard for OpenClaw Gateway. It streams session events (tool calls, LLM responses, tokens) from the Gateway via WebSocket and renders them in a browser SPA.

| Fact | Value |
|------|-------|
| **Stack** | Node.js backend + vanilla JS SPA (no framework) |
| **Binary** | `deepclaw-ui.js` (1838 lines) + `index.html` (3776 lines) |
| **Default port** | `1234` |
| **Default password** | `deepclaw` |
| **Gateway URL** | `ws://127.0.0.1:18789` |
| **Gateway protocol** | v4 |
| **Device identity** | v3 Ed25519 signing (`~/.openclaw/identity/`) |
| **Storage** | JSON files in `data/session-{key}.json` |
| **Auth** | Always enabled (Basic Auth, not conditional) |

## 2. Architecture

```
Gateway (ws://127.0.0.1:18789)
    │ WebSocket (gwSocket)
    ▼
deepclaw-ui.js ──────────────────────────────────────────────────
  │
  ├─ HTTP Server (port 1234)
  │   ├── GET  /                        → index.html
  │   ├── GET  /api/status              → { gatewayReady, sessionCount, ... }
  │   ├── GET  /api/sessions            → session summaries
  │   ├── GET  /api/session/:key        → full session data
  │   ├── GET  /api/events/:key?limit=N → events only
  │   ├── GET  /api/agents              → available agents
  │   ├── POST /api/session/:key/reset  → clear session
  │   ├── POST /api/session/:key/delete → remove session
  │   ├── POST /api/session/:key/clear-events → smart event clear
  │   ├── GET  /api/files/share?path=   → one-shot file token
  │   ├── GET  /api/files/serve?token=  → raw file download
  │   └── GET  /api/files/view?token=   → CodeMirror viewer page
  │
  ├─ Browser WebSocket Server (wss)
  │   └── 17 event types forwarded to browser clients
  │
  └─ SessionState (in-memory Map + disk persistence)
       ├── events[] (max 2000)
       ├── messages[] (max 500)
       ├── tokens{}
       └── disk: data/session-{key}.json
```

**Event flow:** Gateway → deepclaw-ui.js → browser (WebSocket `event.added` + REST fallback)

## 6. Edge Cases & Gotchas

### Dedup

- **Backend message dedup:** `role + '|' + hashString(full_content)` — hash-based, no timestamp. Window: 500 entries, trimmed to 250.
- **Backend event dedup:** `_makeEventKey(sessionId, stream, runId, data)` — prevents duplicate events within a stream.
- **Frontend dedup:** `seenMsgHashes` Set per session + `hasExistingTextEvent()` for streaming dedup.
- **Frontend user_text dedup:** Optimistic messages inserted then deduplicated when gateway echo arrives.

### Session Lifecycle

- `deletedSessions` Set tracks explicitly deleted keys — blocks re-creation from in-flight events.
- `getSession(sk)` defaults `loadFromDisk=true` (not false).
- Sessions load from disk on first access if not in memory.

### Event Conversion

- `stream: 'user'` in `session.tool` → returns `null` (not `user_text`).
- `session.message` with role `user` → `user_text` event.
- `run_end` events are filtered from display in `getFilteredEvents()`.
- Tool call blocks in assistant messages emit `tool_start` + `tool_result` pairs.
- `isFinal`/`isIntermediate` markers on events track streaming completion.

### UI State Machine

- **Chat input:** idle → "Sending..." (pending) → normal. Stop button appears when a request is in flight.
- **Pending/optimistic:** `_pendingRequest` stores `{ sessionKey, text, ts, messageId }`. Gateway echo triggers dedup, not duplication.
- **Scroll:** `userScrolledUp` = true when >80px from bottom. Auto-scroll only when already at bottom.
- **Filters:** `tools` filter = `tool_start` only. `run_end` is always filtered. `llm` filter excludes `run_end` and `run_error`.

### File Sharing

- Tokens expire after 60s if unclaimed.
- Only paths under `$HOME` or `/tmp` are allowed (prefix whitelist).
- Viewer generates full HTML pages with CodeMirror 5 (code) or marked.js (markdown).
- Binary files detected by null-byte check in first 1KB.

## 7. Shell Scripts

### `install.sh` — Interactive setup

One-shot installer that walks through configuration and sets up a systemd user service:

1. **Prompts interactively:** port (default `1234`), password (default `deepclaw`), Gateway WSS (y/N)
2. **Runs `npm install --production`** — installs express-free deps (`ws` only)
3. **Generates `restart.sh`** — hardcodes the chosen env vars into a restart script
4. **Creates systemd user service** → `~/.config/systemd/user/deepclaw-ui.service`
5. **Enables & starts** via `systemctl --user enable --now deepclaw-ui`

### `restart.sh` — Kill & restart

Kills existing `node deepclaw-ui.js` process, then starts fresh with preserved env vars:

```bash
PORT="${PORT:-1234}" \
DCPASS="${DCPASS:-deepclaw}" \
GW_WSS="${GW_WSS:-false}" \
node deepclaw-ui.js
```

After `install.sh` runs, `restart.sh` is regenerated with the user's chosen values hardcoded. ⚠️ After install, `restart.sh` uses `deepclaw` password unless changed at install prompt.

### Systemd user service

```bash
# Status
systemctl --user status deepclaw-ui

# Restart
systemctl --user restart deepclaw-ui

# Stop
systemctl --user stop deepclaw-ui

# Logs
journalctl --user -u deepclaw-ui -f

# Disable on boot
systemctl --user disable deepclaw-ui
```

Service config lives at `~/.config/systemd/user/deepclaw-ui.service` (generated by `install.sh`).

## 8. Common Commands

```bash
# Start server directly (foreground)
node deepclaw-ui.js

# Custom port/password
PORT=3000 DCPASS=mypass node deepclaw-ui.js

# With TLS gateway
GW_WSS=true node deepclaw-ui.js

# Access UI
curl -u :deepclaw http://localhost:1234/

# API calls
curl -u :deepclaw http://localhost:1234/api/status
curl -u :deepclaw http://localhost:1234/api/sessions
curl -u :deepclaw "http://localhost:1234/api/events/agent:main:main?limit=100"
curl -u :deepclaw -X POST http://localhost:1234/api/session/agent:main:main/reset
curl -u :deepclaw -X POST http://localhost:1234/api/session/agent:main:main/delete
```
